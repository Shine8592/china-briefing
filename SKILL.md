# China Briefing — 多尺度中文资讯简报

## 核心能力

**任意尺度，一处生成：**
- 全球 → 区域（亚太/欧洲/中东） → 国家 → 省 → 市 → 区 → 街道→ 社区
- 每层自动选择匹配的搜索源

## 简报结构（固定，不可省略 ⚠️）

> **重要：每个部分都是必填项，禁止省略任何部分！**

```markdown
### 🌍 核心结论 [必填，50-100字，禁止省略]
[1-2句总结本期最重要的信号/趋势/结论]
⚠️ 检查：少于30字 → 退回重写

### 📌 {日期} {地点} 速报 [必填，3条头条，不可少]
**头条1** [150-300字，含嘟嘟点评] ← 少一个都不行
**头条2** [同上]
**头条3** [同上]
⚠️ 检查：每条必须有事实+背景+嘟嘟观点

### 📊 分类热点 [必填，2-4类，每类2-4条]
[每类每条30-80字，不能只有1条]
⚠️ 检查：分类数<2 → 补充相邻主题

### 📈 数据看板 [可选，但建议保留]
[可选：关键数据表格]

### 🎯 行动建议 [必填，非空]
| 优先级 | 行动 | 原因 |
|--------|------|------|
| 高 | ... | ... |
| 中 | ... | ... |
| 低 | ... | ... |
⚠️ 检查：禁止只有"无"或空表
```

## 🚫 反注水规则（强制执行）

生成简报时，以下行为**严格禁止**：

1. **禁止空话套话**
   - ❌ "暂无相关信息"、"暂无数据"
   - ❌ "本文将持续关注"、"敬请期待"
   - ✅ 如果信息不足，用相邻尺度/主题补充

2. **禁止只有标题没有内容**
   - ❌ 头条只有2-3句话
   - ✅ 每条头条必须150-300字（含点评）

3. **禁止重复信息凑字数**
   - ❌ 同一件事换个说法说两遍
   - ✅ 每条信息必须有独特价值

4. **嘟嘟点评不可省略**
   - ❌ 只描述"XX报道了XXX"
   - ✅ 必须加背景、观点、对读者的意义

5. **来源标注完整性**
   - ❌ 来源：未知、来源：网络
   - ✅ 来源：具体媒体名称 + URL

## 🔒 质量门禁（生成后自动检查）

简报生成后，**必须通过以下检查**才能交付：

```python
# 质量门禁函数（在生成脚本中调用）
def quality_gate(briefing_text):
    """生成后强制检查，不达标就打回"""
    errors = []
    
    # 1. 字数检查
    word_count = len(briefing_text)
    if word_count < 800:
        errors.append(f"字数不足（{word_count}/800），重新生成")
    
    # 2. 头条数量检查
    headline_count = briefing_text.count('**头条')
    if headline_count < 3:
        errors.append(f"头条数不足（{headline_count}/3），补充头条")
    
    # 3. 嘟嘟点评检查
    if '嘟嘟点评' not in briefing_text:
        errors.append("缺少嘟嘟点评，补充观点")
    
    # 4. 来源标注检查
    if '来源' not in briefing_text:
        errors.append("缺少来源标注，补充来源")
    
    # 5. 行动建议检查
    if '行动建议' in briefing_text:
        section = briefing_text.split('行动建议')[1][:200]
        if '高' not in section and '中' not in section:
            errors.append("行动建议为空，补充具体建议")
    
    # 6. 注水检查
    water_words = ['暂无', '敬请期待', '持续关注', '暂无数据']
    for ww in water_words:
        if ww in briefing_text:
            errors.append(f"发现注水词「{ww}」，重写该部分")
    
    return errors
```

## 使用流程（标准化，含质量检查）

> **⚠️ 重要：每次生成后必须执行质量检查！**

### 完整流程（7步）

1. **确定尺度和搜索词**
   - 调用 `python3 scripts/briefing_helper_enhanced.py "查询词"` 检测尺度
   - 根据尺度选择对应的搜索词（见下文）

2. **收集信息**
   - 使用 web_search 或 Tavily API 搜索
   - 最少来源数：见"来源多样性规则"表格
   - 每个来源必须保存：标题、URL、内容摘要

3. **生成初稿**
   - 严格按照"简报结构"模板生成
   - 每个部分必须填充，禁止省略
   - 每条头条150-300字（含嘟嘟点评）

4. **🔒 质量门禁检查（强制执行）**
   ```bash
   # 将简报内容保存到临时文件
   echo "简报内容" > /tmp/briefing_check.md
   
   # 执行质量检查
   python3 scripts/quality_gate.py /tmp/briefing_check.md
   ```
   
   - ✅ 如果输出 "✅ 通过"，继续第5步
   - ❌ 如果输出 "❌ 未通过"，查看错误信息，**必须补充/重写**，然后重新检查

5. **补充不达标内容**
   - 根据 quality_gate.py 输出的"修改建议"补充内容
   - 常见补充：
     - 字数不足 → 增加每条头条的背景分析和嘟嘟点评
     - 缺少点评 → 为每条头条添加"嘟嘟点评：..."
     - 来源不足 → 增加搜索词，补充更多来源
     - 行动建议为空 → 根据新闻内容提出具体建议

6. **再次检查**
   - 补充后，再次执行第4步的质量检查
   - 直到输出 "✅ 通过" 且分数 ≥ 80

7. **交付**
   - 输出最终简报
   - 附上质量分数（如："📊 质量分数：85/100"）

## 评论公式

每条头条必须写"嘟嘟点评"，遵守：

```
❌ 只是描述："XX报道了XXX"
✅ 加背景："XX做XXX，结合之前的YYY，说明ZZZ"
✅ 给观点："这意味着...我看好/担心..."
✅ 连点："这和上个月的XX配对，说明趋势在..."
✅ 对读者："对我们来说，意味着..."
```

### 使用方法

#### 1. 干运行模式（推荐先使用）


```bash

```

### 清理级别

| 级别 | 清理内容 | 安全性 |
|------|---------|--------|
| **safe** (默认) | 仅临时文件、编译产物 | ⭐⭐⭐⭐⭐ |
| **moderate** | safe + 旧日志、备份文件 | ⭐⭐⭐⭐ |
| **aggressive** | moderate + 所有缓存 | ⭐⭐⭐ |

### 可清理类型

#### 安全级别（safe）
- `.tmp`, `.temp` 临时文件
- `.swp`, `.swo` Vim交换文件
- `*~` 备份文件
- `.DS_Store`, `Thumbs.db` 系统文件
- `__pycache__/` 目录
- `*.pyc` 编译文件
- `.pytest_cache/` 测试缓存
- `.vscode/`, `.idea/` IDE缓存

#### 中等级别（moderate）
- 包含 safe 级别所有
- `*.bak`, `*.backup` 备份文件（⚠️ 非关键位置）
- 7天前的日志文件

#### 强力级别（aggressive）
- 包含 moderate 级别所有
- 所有 `.log` 文件（非今日）
- 更多缓存文件
